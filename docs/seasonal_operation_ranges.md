# Seasonal Operation Ranges

이 문서는 `gh-01` 300평 연동형 비닐온실에서 `건고추/고춧가루용 적고추`를 운영할 때의 계절별 운전 범위를 정리한다. 목적은 `1.1 계절별 운영 범위 정의`를 고정하고, 이후 정책 엔진과 상태 판단 기준의 상위 입력으로 쓰는 것이다.

## 1. 적용 전제

- 대상 site: `gh-01`
- 시설 형태: `300평 연동형 비닐온실 1동`
- 작형 전제: 봄 정식 후 여름~초가을 수확, 이후 건조/저장 단계로 넘어가는 건고추 작형
- 기준 품종: `왕조`
- 대안 품종: `칼탄열풍`, `조생강탄`

공통 기준:

- 생육 기본 목표: 낮 `25~28℃`, 밤 `18℃ 전후`
- 허용 운전 밴드: 낮 `25~30℃`, 밤 `18~20℃`
- 수정 불량 경계: `30℃ 초과` 또는 `13℃ 이하`
- 정식기 보수 기준: 밤 `16℃ 이상`

## 2. 계절별 운영 범위

### 겨울 `12~2월`

- 주 운영 단계: 육묘, 정식 전 준비, 난방/보온 중심
- 목표:
  - 발아: `28~30℃`
  - 육묘: 낮 `22~30℃`, 밤 `15℃ 이상`
  - 순화: 정식 `7~10일 전` 낮 `22~23℃`, 밤 `14~15℃`
- 운영 포인트:
  - 야간 최저온 유지 실패를 최우선 리스크로 본다.
  - 보온커튼, 난방기, 지온 보전을 우선 사용한다.
  - 과습보다 저온과 광 부족 리스크를 먼저 본다.

### 봄 `3~5월`

- 주 운영 단계: 정식, 활착, 초기 영양생장, 개화 진입
- 목표:
  - 낮 `25~28℃`
  - 밤 `16~18℃`
- 운영 포인트:
  - 활착 전까지 배지 과습과 저온을 동시에 피한다.
  - 낮 일사 확보를 우선하되, 급격한 기온 상승 시 환기를 점진적으로 연다.
  - 개화기에는 오전 광 확보와 꽃가루 활력 유지가 핵심이다.

### 여름 `6~8월`

- 주 운영 단계: 개화/착과, 과실 비대, 홍고추 수확
- 목표:
  - 낮 `25~28℃` 유지 시도
  - 실제 제어 경계: `30℃ 이하` 유지 우선
  - 밤 `18~22℃`
- 운영 포인트:
  - 환기창, 순환팬, 차광을 함께 써서 `30℃ 초과 시간`을 최소화한다.
  - 고온기에는 낙화, 웃자람, 일소, 배지 과건조를 동시 관리한다.
  - 배액률과 배지 함수율 회복 속도를 관수 판단의 핵심 지표로 둔다.

### 가을 `9~11월`

- 주 운영 단계: 후기 수확, 작기 종료, 건조/저장 전환
- 목표:
  - 낮 `22~28℃`
  - 밤 `16~18℃`
- 운영 포인트:
  - 야간 저온, 결로, 병해 잔존 리스크를 우선 본다.
  - 후기에는 신규 착과보다 기존 과실의 착색과 수확 품질 확보를 우선한다.
  - 첫서리 예보 이후에는 추가 착과보다 잔존 과실 회수와 철거 판단으로 전환한다.

## 3. 정책 연결 원칙

- `winter`: 난방/보온 우선, 야간 저온 hard watch
- `spring`: 활착 우선, 저온 + 과습 동시 경계
- `summer`: 고온 억제 우선, `30℃ 초과 누적시간` 관리
- `autumn`: 저온·결로·병해 관리와 잔존 과실 회수 우선

계절 값은 달력만으로 결정하지 않고 아래 입력을 함께 본다.

- 현재 생육 단계
- 최근 7일 외기 패턴
- 정식 후 경과일
- 수확/철거 여부

## 4. 후속 반영 항목

- 계절별 setpoint를 `policy-engine` 정책 JSON에 반영
- 계절별 평가셋 slice 추가
- 품종별 계절 민감도 차이를 `cultivar` metadata에 연결
