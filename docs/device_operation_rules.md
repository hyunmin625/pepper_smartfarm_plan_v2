# Device Operation Rules

이 문서는 `2.1 장치 운전 경험 규칙 정리` 결과를 고정한다. 목적은 공식 재배 지식, 현장 사례, 안전 규칙을 묶어 장치 운전의 기본 SOP를 정의하는 것이다.

## 적용 원칙

- 대상 site: `gh-01`
- 기준 품종: `왕조`
- 장치 명령은 항상 `sensor quality -> policy -> execution-gateway -> plc-adapter` 순서로 검증한다.
- 단일 장치만 보지 않고 `온실 환경 + 외기 + 근권 + 최근 실행 결과`를 함께 본다.
- `recommended_max`를 넘는 조작은 승인 경로로 보낸다.

## 장치별 운전 규칙

### 1. 환기창 + 순환팬

- 여름철 고온 억제는 `환기창 -> 순환팬 -> 제한 차광` 순서로 우선 적용한다.
- 외기 냉방 효과가 있고 `wind_lock`이 없으면 환기창을 먼저 연다.
- 순환팬은 환기와 함께 써서 상하층 온도 편차와 정체 공기를 줄인다.
- 환기창이 이미 충분히 열렸는데도 고온이 지속되면 차광을 추가 검토한다.

### 2. 차광커튼

- 전체 강차광을 먼저 거는 방식은 피한다.
- 고온 완화가 필요할 때는 `부분 차광` 또는 `낮은 차광률`을 우선한다.
- 장마기 저일조와 겹치면 차광률을 더 낮추고 환기/습도 관리를 우선한다.
- `position_pct 100`은 강광 또는 비상 상황으로 한정한다.

### 3. 관수 밸브

- 관수는 `긴 1회 급수`보다 `짧은 펄스 + 반응 확인`을 기본으로 둔다.
- 다음 조건을 함께 보고 관수한다: 배지 함수율, 관수 후 회복 속도, 배액률, 급배액 EC 차이.
- 고온기에는 1회 길이를 크게 늘리기보다 빈도를 늘린다.
- `line_pressure_fault` 또는 readback mismatch가 있으면 반복 명령보다 수동 점검을 우선한다.

### 4. 양액기

- 양액기는 `recipe_id`와 `mix_volume_l`를 동시에 큰 폭으로 바꾸지 않는다.
- 공급 EC/pH와 배액 EC/pH를 같은 블록 기준으로 비교해 drift를 먼저 확인한다.
- `flush`와 `sanitize`는 일반 생육 recipe와 분리해 승인 또는 작업자 개입 하에 실행한다.
- 센서 품질이 불안정하면 recipe 변경보다 검교정과 mixer 점검을 우선한다.

### 5. 난방기

- 난방은 야간 최저온 방어가 1순위다.
- 정식기에는 밤 `16℃ 이상`, 일반 야간 운전은 밤 `18℃ 전후`를 우선 목표로 둔다.
- `stage 2`는 혹한, 저온 지속, 승인된 복구 상황에서만 사용한다.
- 과습 상태에서 난방만 올리는 조작은 결로/병해 위험을 함께 검토한다.

### 6. CO2 주입기

- CO2는 환기창이 많이 열려 있거나 `vent_open_lock`이 active면 주입하지 않는다.
- 오전 광이 확보되고 환기 손실이 낮을 때 우선 사용한다.
- `dose_pct`를 크게 올리기 전에 실제 CO2 회복 속도와 환기 상태를 본다.
- `co2_high_lock` 또는 센서 품질 불량이면 즉시 차단한다.

### 7. 제습기 + 건조실 팬

- 건조실은 `팬 유량 + 제습`의 조합으로 보고, 제습기만 단독으로 세게 올리지 않는다.
- 함수율 저하 속도가 느리면 팬 유량과 공기 교환을 먼저 확인한다.
- 과건조가 시작되면 팬 출력을 먼저 낮추고, 필요 시 제습 단계도 함께 낮춘다.
- 건조실 고온 프로파일 전환은 승인 액션으로 본다.

### 8. 원수 밸브

- 원수 밸브는 양액기/관수 경로의 선행 장치로 취급한다.
- `low_pressure_lock` 또는 공급 이상이 있으면 후속 급수 명령도 함께 block한다.
- 원수 라인 점검 전에는 `open` 상태 유지보다 `closed` 기본을 우선한다.

## 공통 금지 패턴

- 센서 품질이 `bad`인데 장치 setpoint를 확대하는 조작
- readback mismatch 상태에서 같은 명령을 연속 재발행하는 조작
- `manual_override`, `estop`, `safe_mode` active 상태에서 자동 제어 명령 발행
- 환기 손실이 큰 상태에서 CO2를 올리는 조작
- 저일조/장마기에 강차광을 장시간 유지하는 조작

## 후속 반영

1. `policy-engine`의 hard block / approval 정책 JSON으로 변환
2. action recommendation seed와 eval case에 장치 운전 SOP 반영
3. 운영 로그가 쌓이면 `farm_case`와 비교해 규칙을 보정
